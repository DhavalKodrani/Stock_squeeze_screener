GitHub token

"Refresh Now" triggers a fresh GitHub Actions run and reads back the result, so it needs a token with access to this repo. It is stored only in your browser's local storage and is never committed anywhere — it's sent only to GitHub's API.

Prerequisite: your GitHub account must have access to this repository — ask the repo owner to add you as a collaborator (and accept the email invite). Without that, GitHub rejects scan requests with HTTP 403 no matter what token you use. Viewing results needs no token and no access.

Then create a classic personal access token (fine-grained tokens are unreliable for triggering workflow runs via the API, even with the right permissions set):

  1. Go to github.com/settings/tokens/new
  2. Give it a name and expiration
  3. Check the scopes: repo and workflow
  4. Generate, then paste it below
